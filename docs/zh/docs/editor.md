# 编辑器

编辑器是 NimoteCode 工作区的中心。无论文件来自本地项目还是 SSH 远程工作区，最终都会在这里进入实际修改流程。

## 核心编辑流程

1. 从本地资源管理器或远程资源管理器打开工作区。
2. 在资源管理器或搜索结果中选择文件。
3. 在标签化编辑器中修改内容。
4. 保存文件。
5. 继续进入终端、Git、AI、LSP 或调试面板完成下一步。

## 主要能力

- 支持多个打开中的编辑标签页。
- 支持光标行列跟踪。
- 支持复制、剪切、粘贴、全选、撤销与重做。
- 支持将当前文件作为 AI Chat 上下文。
- 在存在符号数据时，可通过大纲面板进行结构导航。
- 提供面向编程场景的移动键盘控制。
- 可与 AI、大纲、终端等面板形成联动工作流。

## 实用建议

- 向 AI 提问前尽量保持目标文件处于激活状态，便于助手使用当前文件上下文。
- 在远程项目中可先用搜索定位内容，再进入编辑器修改。
- 保存后再进入 Source Control 审查变更，会更接近真实提交流程。
